Creating a static website using Hugo

This section will provide instructions on setting up and creating a statically generated website using Hugo.

Andrew Etter details the importance of building a website as a means of distributing any technical writing or documentation. This is because websites never become outdated in the way that traditional documents like PDFs do. Websites can easily be edited with new information after they have already been published, and static website generators make the process of creating a website quick and seamless.

  1. Install Hugo on your device. You can find specific download methods for your operating system on their website.

    • Note: You will also need to have Git installed on your device if it isn’t already.

  2. Verify that Hugo has been installed correctly using the following command: hugo version

  3. Create a new Hugo project using the following command: hugo new project project_name

    • Replace ‘project_name’ with what you would like to name your project

  4. Change into the directory of the newly created project using the following command in your terminal with your project’s name: cd project_name

  5. Initalize a Git repository within the project repo using the following command: git init

  6. (Optional) Install a theme that can be used to personalize your static website by cloning a submodule

    • e.g. git submodule add https://github.com/hanwenguo/hugo-theme-nostyleplease.git will install the nostyleplease theme used for this static website
    • Add the following line to the hugo.toml file in the root directory of the project theme = "nostyleplease"

Formatting a resume using Markdown

Now that you’ve generated a static website project using Hugo, the next step is to create a piece of content that you will provide to your website. This content, specifically known as a page will store the contents of your resume. The most popular format for writing content is Markdown, which is a type of lightweight markup language.

Andrew Etter elaborates on why lightweight markup is the superior choice for formatting content on a website in his book. Notably, Etter states that lightweight markup languages provide an element of human-readability that raw languages such as XML, or HTML don’t. Lightweight markup languages allow the writer to create a human readable document in a language such as Markdown, and that will then be convereted into the raw XML/HTML by the static website generator you use.
